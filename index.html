✦ Step-by-Step Guide

01

Launch EC2 Instance

Create your virtual server on AWS with the right configuration.

  1. Log into the AWS Management Console → navigate to EC2.
  2. Click "Launch Instance" and give it a name (e.g., my-cloud-server).
  3. Select Ubuntu Server 22.04 LTS (Free Tier eligible) as the AMI.
  4. Choose instance type: t2.micro (Free Tier).
  5. Under Key Pair, click "Create new key pair" → name it my-key → Download my-key.pem.
  6. Under Network Settings, allow SSH (port 22) from your IP.
  7. Click Launch Instance. ✅
aws cli alternative
# Optional: Launch via AWS CLI
aws ec2 run-instances \
  --image-id ami-0c02fb55956c7d316 \
  --instance-type t2.micro \
  --key-name my-key \
  --security-group-ids sg-XXXXXXXX \
  --count 1
02

Connect via SSH

Use your key pair to securely log into the server.

  1. Find your instance's Public IPv4 address in the EC2 dashboard.
  2. Open your terminal and navigate to where my-key.pem was downloaded.
  3. Set correct permissions on the key file.
  4. Connect using the SSH command below.
bash
# Set permissions (required — SSH rejects open key files)
chmod 400 my-key.pem

# Connect to your EC2 instance (replace with your IP)
ssh -i "my-key.pem" ubuntu@<YOUR-PUBLIC-IP>

# You should see the Ubuntu welcome banner!
Welcome to Ubuntu 22.04.3 LTS (GNU/Linux 6.2.0 x86_64)
03

Install Nginx & Node.js

Set up the web server and JavaScript runtime on your fresh Ubuntu instance.

  1. Update the package list and upgrade existing packages.
  2. Install Nginx as the reverse proxy / web server.
  3. Install Node.js 20 using NodeSource's setup script.
  4. Install PM2 — a process manager that keeps your Node app running.
bash — on EC2
# Update system packages
sudo apt update && sudo apt upgrade -y

# Install Nginx
sudo apt install nginx -y
sudo systemctl start nginx
sudo systemctl enable nginx

# Install Node.js 20 via NodeSource
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install nodejs -y

# Verify versions
node -v   # v20.x.x
npm -v    # 10.x.x

# Install PM2 globally
sudo npm install -g pm2
04

Deploy Your Node.js App

Write, run, and proxy a simple Express app to serve over the web.

  1. Create a project directory and initialize it.
  2. Write a minimal Express server that responds with "Hello from the cloud!".
  3. Start the app with PM2 so it stays alive after logout.
  4. Configure Nginx to proxy requests from port 80 → port 3000.
bash — on EC2
# Create project folder
mkdir ~/myapp && cd ~/myapp
npm init -y
npm install express

# Create app entry point
cat > server.js << 'EOF'
const express = require('express');
const app = express();
const PORT = 3000;

app.get('/', (req, res) => {
  res.send('<h1>Hello from the Cloud! ☁️</h1><p>Deployed on AWS EC2</p>');
});

app.listen(PORT, () => {
  console.log(`Server running on port ${PORT}`);
});
EOF

# Start with PM2
pm2 start server.js --name "myapp"
pm2 startup        # auto-start on reboot
pm2 save

# Configure Nginx reverse proxy
sudo nano /etc/nginx/sites-available/myapp
📄 /etc/nginx/sites-available/myapp
server {
    listen 80;
    server_name <YOUR-PUBLIC-IP>;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
bash — enable the site
# Enable site and reload Nginx
sudo ln -s /etc/nginx/sites-available/myapp /etc/nginx/sites-enabled/
sudo nginx -t        # test config — should say OK
sudo systemctl reload nginx
05

Open Security Group Ports

Configure AWS firewall rules to allow public web traffic to your server.

  1. In the AWS Console, go to EC2 → Security Groups.
  2. Select the security group attached to your instance.
  3. Click Edit Inbound Rules and add the following rules.
  4. Save the rules — traffic is now allowed.
TypeProtocolPortSourcePurpose
SSHTCP22My IPSecure shell access
HTTPTCP800.0.0.0/0Public web traffic
HTTPSTCP4430.0.0.0/0Secure web traffic (future)
CustomTCP3000My IPDirect Node.js access (dev)
aws cli alternative
# Open port 80 via CLI
aws ec2 authorize-security-group-ingress \
  --group-id sg-XXXXXXXX \
  --protocol tcp \
  --port 80 \
  --cidr 0.0.0.0/0

# Check UFW firewall on the server itself
sudo ufw allow 'Nginx Full'
sudo ufw allow OpenSSH
sudo ufw enable
sudo ufw status
🎉 Your app is live! Open a browser and visit http://<YOUR-PUBLIC-IP> — you should see your Node.js app served over the internet.

# ☁️ Cloud Virtual Machine Setup — AWS EC2

A hands-on project demonstrating how to launch, configure,
and deploy a web application on a real cloud server.

## 🛠️ Tech Stack
- **Cloud**: AWS EC2 (t2.micro, Free Tier)
- **OS**: Ubuntu 22.04 LTS
- **Web Server**: Nginx (reverse proxy)
- **Runtime**: Node.js 20 + Express
- **Process Manager**: PM2
- **Firewall**: AWS Security Groups + UFW

## 🚀 What I Built
A cloud-hosted Node.js Express application accessible
via public IP, served through Nginx as a reverse proxy,
and managed by PM2 for reliability.

## 📚 Skills Demonstrated
- SSH key-pair authentication
- Linux server administration
- Nginx configuration & reverse proxying
- AWS Security Group firewall rules
- Process management with PM2
- Cloud infrastructure setup from scratch
